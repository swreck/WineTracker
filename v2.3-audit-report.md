### Test 14: Whitespace-only chat message
**Action:** POST /remi/chat with message "   " (three spaces)
**Result:** `{"error":"Remi is unavailable right now"}` (500)
**Verdict:** FAIL — should return 400 with "message is required" (same as empty string), not 500.
**Note:** Empty string `""` correctly returns 400. The validation checks for falsy/empty but `"   "` is truthy in JS, passes validation, then likely causes an error downstream (possibly the Anthropic API rejecting empty-after-trim content).
**Fix:** Add `.trim()` to the message validation check.
