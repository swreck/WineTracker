# Wine Tracker 2.2 — Chaos Audit Round 3

**Date:** 2026-03-31
**Environment:** https://wine-tracker-backend-production.up.railway.app/api
**Method:** curl-based API testing, sequential execution
**Wine count at start:** 171 (test wine created and cleaned up during audit)

---

## Bugs

### BUG-1: Chat focusWineId context leakage (INTERMITTENT, SERIOUS)
**Test 3:** Opened chat with focusWineId=532 (CHAOS TEST Tempranillo), then immediately opened chat with focusWineId=131 (PATZ & HALL SC CHARD). Remi responded about the Tempranillo — called it "a red wine, Tempranillo" when asked about wine 131, which is a white Chardonnay. On the third attempt with the same focusWineId=131, Remi correctly identified it as Patz & Hall white Chardonnay. **Conversation state is leaking across calls when focusWineId changes.**

**Impact:** User opens wine A, chats, goes back to list, opens wine B, chats — Remi talks about wine A. This would be extremely confusing on a phone.

### BUG-2: Chat with non-existent focusWineId returns wrong wine (SERIOUS)
**Test 12:** `POST /remi/chat` with `focusWineId: 999999` and message "Tell me about this wine" — instead of returning an error, Remi responded with detailed info about Vietti Barbera d'Alba 2018 (id=182). No indication that the requested wine doesn't exist.

**Impact:** If a wine is deleted while the user has it open, they get phantom information about a random other wine.

### BUG-3: Import silently drops rating and notes
**Test 1:** Imported via editedBatches with `rating: 7.5` and `notes: "Test wine for chaos audit round 3"`. Import succeeded (winesCreated=1, vintagesCreated=1) but `tastingsCreated: 0`. The rating and notes were accepted without error but never persisted. Checking the wine detail confirmed zero tasting events.

**Impact:** User imports wines with ratings from a receipt — ratings silently vanish. Data loss without any feedback.

### BUG-4: Dismiss non-existent suggestion returns success
**Test 14:** `DELETE /remi/suggestions/999999` returns `{"dismissed": true}` with HTTP 200. Should return 404.

**Impact:** Low severity, but incorrect API semantics. Could mask bugs in the frontend dismiss logic.

## UX Friction

### UXF-1: Enrich non-existent wine returns 200 with error in content
**Test 17:** `POST /remi/enrich` with `wineId: 999999` returns HTTP 200 with `{"profile": "Wine not found in collection."}`. The error message is embedded in the profile text field rather than returned as a proper error with 404 status.

**Impact:** Frontend would render "Wine not found in collection." as the enrichment paragraph instead of showing an error state.

### UXF-2: Dismissed wine suggestions can resurface with new IDs
**Test 4:** Dismissed 3 suggestions (ids 7-9, including Rioja Alta Ardanza wineId=24). After regenerating, Rioja Alta Ardanza reappeared as id=14 with slightly different wording. Dismissal is per-suggestion-ID, not per-wine.

**Impact:** User swipes away a suggestion they're not interested in; it comes back next refresh. Mildly annoying but not broken.

### UXF-3: Remi too cautious on general recommendations
**Test 7:** Asked "What wines should I drink this week?" — Remi asked for clarification instead of proactively recommending. Required a second, more explicit prompt ("Just pick 3 wines") to get actual recommendations. The follow-up response was excellent.

**Impact:** Extra round-trip on the most natural question a user would ask. Remi should default to making bold picks when asked open-ended questions.

### UXF-4: Themes endpoint is slow (20 seconds)
**Test 21:** Themes generation took 19.8 seconds. On a phone, that's an eternity with no intermediate feedback.

**Impact:** User may think the app crashed. Needs loading state in the UI at minimum; caching or streaming would be better.

### UXF-5: No rate limiting on suggestion generation
**Test 13:** Two rapid POST /remi/suggestions calls both succeed, creating 12 new suggestions (ids 19-30). Previous suggestions (13-18) are presumably orphaned in the DB.

**Impact:** Accidental double-taps on "refresh suggestions" burn API credits and leave orphaned records.

## Priority Fix List

1. **BUG-1 (focusWineId context leakage)** — Most impactful. Users will hit this constantly when browsing wines and chatting.
2. **BUG-2 (non-existent focusWineId returns wrong wine)** — Validate wine exists before sending to AI; return 404 if not.
3. **BUG-3 (import drops rating/notes)** — Either create tasting events from import data, or strip rating/notes from the import schema so users don't think they're being saved.
4. **UXF-1 (enrich error in content)** — Return proper 404 instead of 200 with error text in profile field.
5. **UXF-4 (themes performance)** — Add caching, streaming, or at minimum a clear loading state.
